# Support WFGY

WFGY is an open, MIT-licensed ecosystem for reasoning, debugging, evaluation, and structural AI system analysis.

If this project has helped you debug pipelines, design safer reasoning flows, or think more clearly about structural AI problems, you can support its continued development below.

If you are looking for a structured pilot, audit, or collaboration path, do not use this page as the main entry.  
Start with [Work with WFGY](./WORK_WITH_WFGY.md).

If you want to understand the broader public context first, see:

* [Adopters](./ADOPTERS.md)
* [Case Evidence](./CASE_EVIDENCE.md)
* [Evidence Timeline](./EVIDENCE_TIMELINE.md)
* [Ecosystem Map](./ECOSYSTEM_MAP.md)

## What this page is

This page is the support path for the public WFGY ecosystem.

Its purpose is simple:

help people support the continued development, maintenance, documentation, and public evolution of WFGY.

This page is not:

* a consulting menu
* a pilot offer
* a guarantee of custom implementation
* a shortcut to structured collaboration
* a feature purchase page

Support helps sustain public work.  
It does not automatically create a scoped project relationship.

---

## Support and collaboration are different

This distinction matters.

### Support

Support means helping sustain the public ecosystem.

That includes work such as:

* maintaining and improving documentation
* refining ecosystem structure and navigation
* sustaining public proof and evidence pages
* expanding diagnostic and teaching materials
* continuing framework and application development

### Collaboration

Collaboration means a more structured working relationship around a specific problem, pilot, audit, or research-facing use case.

If you want that, start here instead:

* [Work with WFGY](./WORK_WITH_WFGY.md)
* [Pilot Offer One-Pager](./PILOT_OFFER_ONE_PAGER.md)
* [Sample Deliverable](./SAMPLE_DELIVERABLE.md)

If you want to help keep the public system alive, use the support path.  
If you want scoped work, use the collaboration path.

---

## Why support matters

WFGY is not only a code repository.

It is also a growing public system of:

* framework layers
* diagnostic maps
* public documentation
* proof and evidence surfaces
* contributor workflows
* collaboration entry points
* evaluation and application-facing materials

Keeping these surfaces coherent takes time.

Support helps sustain work such as:

* writing and refining public docs
* improving routing and readability
* maintaining recognition and evidence pages
* extending the Problem Map line and related diagnostic surfaces
* improving contribution workflows
* continuing framework and application development

Public infrastructure work often creates value before it creates cash flow.  
Support helps close that gap.

## What support helps fund

Support makes it easier to continue work such as:

* maintaining WFGY core documentation
* improving Problem Map and related diagnostic surfaces
* expanding public proof pages
* refining ecosystem structure and readability
* preparing reproducible evaluation and pilot materials
* sustaining public work that would otherwise slow down

In short, support helps keep the ecosystem usable, not just visible.

---

## What support does not buy

To keep boundaries clear, support should not be read as automatically purchasing any of the following:

* private consulting
* guaranteed feature work
* guaranteed review priority
* guaranteed custom debugging
* guaranteed implementation help
* pilot or audit engagement
* deployment or integration support

If you need those things, use the collaboration path instead.

That boundary protects both sides.

If you would like any of the following:

* confirmation of receipt
* public acknowledgment as a supporter
* to share feedback or suggest focus areas

**donate@onestardao.com**

* transaction hash
* network used
* optional message

There is no guaranteed feature exchange, but thoughtful feedback from supporters can meaningfully influence project direction.
